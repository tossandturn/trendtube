## 7. Content Refresh Schedule

### 7.1 Refresh Frequency by Page Type

| Page Type | Refresh Frequency | Signals That Trigger Update | Auto vs Manual |
|-----------|-------------------|----------------------------|----------------|
| **Trend pages** (`/trends/*`) | **Weekly** | New trending data, seasonal shifts, viral events | Auto (data pipeline) |
| **Competitor alternatives** (`/alternatives/*`) | **Monthly** | Competitor pricing changes, new features, user reviews | Semi-auto (monitoring alerts) |
| **Niche pages** (`/niches/*`) | **Quarterly** | CPM shifts, saturation changes, algorithm updates | Semi-auto (quarterly data refresh) |
| **Tool pages** (`/tools/*`) | **Monthly** | Feature releases, UI changes, new integrations | Manual (product team) |
| **Algorithm reports** (`/algorithm/*`) | **As needed** | YouTube algorithm updates, official announcements | Manual (editorial team) |
| **Data studies** (`/reports/*`) | **Annually** | New annual data collection | Manual (research team) |

### 7.2 Automated Refresh Signals

Implement monitoring for:

```yaml
signals:
  trend_velocity_change:
    threshold: ±30% week-over-week
    action: Flag for content review
    
  competitor_feature_release:
    source: RSS/API monitoring
    action: Update comparison tables within 48 hours
    
  search_ranking_drop:
    threshold: Drop >5 positions for target keyword
    action: Trigger content audit + refresh
    
  seasonal_event:
    examples: [Christmas, Black Friday, Summer holidays]
    action: Pre-publish seasonal trend content 2 weeks ahead
    
  youtube_algorithm_update:
    source: YouTube Creator Insider, official blog
    action: Publish algorithm analysis within 24 hours
    
  user_feedback:
    source: On-page feedback widget, support tickets
    action: Aggregate monthly, prioritize high-complaint pages
```

### 7.3 Refresh Workflow

```
1. Signal Detection → Monitoring system flags page
2. Priority Assignment → P0 (24h), P1 (1 week), P2 (1 month)
3. Content Audit → Check accuracy, freshness, ranking position
4. Update Execution → Edit programmatic data or editorial layer
5. Re-index Request → Submit to Google Search Console
6. Validation → Check ranking recovery within 2 weeks
7. Log Update → Record in content calendar
```

| Metric | Without Refresh | With Refresh | Improvement |
|--------|---------------|--------------|-------------|
| Average page age | 6+ months | <30 days | 5x fresher |
| Ranking decay rate | 15% monthly | 5% monthly | 3x slower |
| Trend page traffic | Declining after 2 weeks | Stable/growing | +40% retention |
